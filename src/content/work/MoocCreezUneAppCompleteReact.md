##### Présentation du projet

Dans le cadre de mon apprentissage personnel, j’ai suivi le MOOC **« Créez une application React complète »** proposé par OpenClassrooms.  
Ce parcours m’a permis de concevoir une **Single Page Application (SPA) robuste**, en appliquant les bonnes pratiques modernes de l’écosystème React, aussi bien sur l’architecture, la gestion des données, le styling que les tests.

L’objectif était de se rapprocher d’un **niveau de développement professionnel**, en construisant une application maintenable, évolutive et testée, tout en approfondissant la compréhension interne de React.

---

##### Architecture et structuration de l’application

Une attention particulière a été portée à la **structure du projet** afin de garantir lisibilité et évolutivité :

- Mise en place d’une **architecture modulaire**
- Séparation claire des responsabilités (pages, composants, hooks, services)
- Organisation cohérente des fichiers et dossiers
- Conception d’une **Single Page Application (SPA)**

La navigation est gérée via **React Router**, permettant :
- une navigation fluide sans rechargement de page
- une gestion claire des routes et des vues
- une expérience utilisateur proche d’une application native

---

##### Gestion des composants et typage

- Utilisation des **props** pour la communication entre composants
- Validation des données reçues avec **PropTypes**
- Conception de composants réutilisables et isolés
- Compréhension de l’importance du typage pour la maintenabilité du code

---

##### Styling et UI

Le style de l’application est géré avec **styled-components**, permettant :

- un **scoping du CSS** au niveau des composants
- une meilleure lisibilité des styles
- la création de composants visuels réutilisables
- une séparation claire entre logique et présentation

Cette approche améliore la maintenabilité et limite les conflits de styles à grande échelle.

---

##### Gestion des données avec les hooks

L’application exploite pleinement les **hooks React** pour gérer les données et les effets :

- `useState` pour la gestion de l’état local
- `useEffect` pour les effets secondaires
- Réalisation de **calls API** et affichage dynamique des données
- Gestion des états de chargement et d’erreur

---

##### Hooks avancés et hooks personnalisés

Afin d’aller plus loin dans la logique applicative :

- Création de **hooks personnalisés** pour factoriser la logique
- Réutilisation de comportements complexes entre composants
- Amélioration de la lisibilité et de la maintenabilité du code
- Compréhension avancée du cycle de vie des composants

---

##### Partage de données globales

L’application met en place un **state global** via :

- le **Context API**
- le hook `useContext`

Cela permet :
- de partager des données entre plusieurs composants sans prop drilling
- de centraliser certaines logiques applicatives
- d’améliorer la scalabilité de l’application

---

##### Tests et qualité du code

Une partie importante du projet est dédiée aux **tests** :

- Mise en place de tests unitaires avec **Jest**
- Tests de composants avec **React Testing Library**
- Vérification du rendu, des interactions utilisateur et des comportements
- Approfondissement des bonnes pratiques de test en React

Cette approche garantit une application plus fiable et plus facile à faire évoluer.

---

##### Évolution dans l’écosystème React

Le cours aborde également l’évolution de React :

- Compréhension des **anciennes syntaxes** basées sur les composants classe
- Affichage de données d’une API dans un **composant classe**
- Comparaison entre composants classe et composants fonctionnels
- Meilleure compréhension de l’histoire et de l’évolution de React
